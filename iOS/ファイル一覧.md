# 双極AI iOSアプリ ファイル一覧

## ファイル構成

```
iOS/
├── Models/
│   ├── DailyLog.swift              # 日次ログデータモデル
│   ├── CalculationResult.swift     # 計算結果データモデル
│   └── HealthKitData.swift         # HealthKitデータモデル
├── Services/
│   ├── GASService.swift            # GAS API通信サービス
│   ├── HealthKitService.swift      # HealthKit読み取りサービス（基本構造）
│   └── LineNotifyService.swift     # LINE Notify APIサービス
├── Views/
│   ├── InputView.swift             # 入力画面（Mood + 定型質問4本）
│   ├── ResultView.swift            # 結果表示画面
│   ├── SettingsView.swift          # 設定画面（LINE Notifyトークン）
│   ├── RebootView.swift            # Rebootプログラム画面
│   └── Components/
│       └── StageButton.swift       # ステージ選択ボタン
├── Utils/
│   └── Constants.swift             # 定数（API URLなど）
├── BipolarAIApp.swift              # アプリエントリーポイント
├── ContentView.swift               # メイン画面（TabView）
└── ドキュメント/
    ├── README.md                   # プロジェクト概要
    ├── 実装ガイド.md               # 実装手順
    ├── HealthKit実装詳細.md        # HealthKit実装の詳細
    ├── Mac環境での作業チェックリスト.md  # Mac環境での作業手順
    ├── まとめて作業する手順.md     # 効率的な作業手順
    └── 完成状況まとめ.md           # 完成状況のまとめ
```

## ファイルの説明

### Models（データモデル）

- **DailyLog.swift**: GASエンドポイントに送信する日次ログデータ
- **CalculationResult.swift**: GASエンドポイントから返ってくる計算結果
- **HealthKitData.swift**: HealthKitから取得するデータ

### Services（サービス層）

- **GASService.swift**: GAS APIとの通信を担当
- **HealthKitService.swift**: HealthKitからのデータ読み取りを担当（実装が必要）
- **LineNotifyService.swift**: LINE Notify APIとの通信を担当

### Views（UI画面）

- **InputView.swift**: メインの入力画面
- **ResultView.swift**: 計算結果の表示画面
- **SettingsView.swift**: 設定画面（LINE Notifyトークン入力）
- **RebootView.swift**: Rebootプログラム画面
- **Components/StageButton.swift**: ステージ選択ボタンコンポーネント

### Utils（ユーティリティ）

- **Constants.swift**: アプリ定数（GASエンドポイントURLなど）

### App（アプリ統合）

- **BipolarAIApp.swift**: アプリのエントリーポイント
- **ContentView.swift**: メイン画面（TabViewで入力と設定を切り替え）

## Mac環境での作業

すべてのファイルが準備できているので、Mac環境でXcodeプロジェクトを作成して、これらのファイルをコピーするだけです。

詳細は `Mac環境での作業チェックリスト.md` を参照してください。
